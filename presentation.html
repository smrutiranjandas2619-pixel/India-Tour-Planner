Introduction

Project Overview & Objectives

The Fragmented Planning Problem

Travelers face hours of friction cross-referencing multiple sites to build a travel docket: looking up sights, calculating dynamic budgets, finding accommodations, tracking weather forecasts, and scraping regional safety alerts.

The Unified Intelligent Solution

India Tour Planner merges these layers. It creates a cohesive web dashboard that generates structured dossiers instantly, provides interactive path mapping, outputs tailored expenses, and offers a conversational chatbot agent.

AI Integration

Retrieval-Augmented Generation (RAG)

The Semantic Context Loop

  • Local Vector DB: Stores high-quality textual guides on Indian cities and monuments.
  • Cosine Search: Calculates matching chunks dynamically via NumPy calculations.
  • Prompt Coupling: Combines retrieved information, user specifications, weather data, and coordinate lists before LLM generation.

Dual Model Optimization

We strategically map user requests to appropriate LLM architectures to optimize speed and structure:

Gemini 1.5 Flash: Excellent handling of complex structured inputs, rendering structured JSON output for itinerary coordinates.
Llama 3.3 (Groq): Leveraged for high-speed conversational Q&A back-and-forths inside the advisor chatbox.

Database

Data Schema & Storage Architecture

Relational Database Structure (SQLite)

  • Users Schema: Houses verified profile details, salt/hash password fields, and validation statuses.
  • Trips Schema: Stores foreign-key mapping to users, destination strings, travel dates, budget flags, and the generated JSON itinerary.
  • Coordinate Cache Schema: Stores maps of place names to lat/long vectors, avoiding redundant third-party API coordinate checks.

Vector Database File Structure

  • Vector File: Stored locally as a lightweight JSON document containing vectorized paragraphs.
  • Local Matching: Avoids expensive external database hosting, performing quick calculations in local memory.
  • Content Sync: Can be rebuilt or updated incrementally via administrative python utilities.

DevOps

Unified Deployment & DevOps

Combined Monorepo Strategy

Rather than deploying backend and frontend systems separately, the project implements a single-repository compile command. During deployment:

  • Node packages are installed to build the React package.
  • Static assets are generated and moved into the backend.
  • FastAPI mounts the directory and serves it directly.

Clean Reverse Proxy & CORS

By routing the web client directly from the API port in production, we eliminate CORS issues:

  • Development: Vite proxies api calls to port 8000.
  • Production: Requests to `/` serve the static React file while `/api/` endpoints are served by FastAPI.
  • Maintains full scalability under zero hosting expenses.
